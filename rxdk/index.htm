RXDK Extension Documentation

Documentation for the RXDK VS Code extension — creating, building, deploying, running, and debugging original Xbox titles from VS Code. For the Xbox SDK API reference (Direct3D 8, XAPI, DirectSound, …) use RXDK: Xbox SDK Documentation instead; that is a separate reference set in the same viewer.

Contents

  • Getting started — prerequisites, create a project, build/deploy/run/debug.
  • Project manifest (rxdk.project.json) — complete property reference.
  • Debugging — breakpoints, Locals, the Globals pane + visibility toggle, Watch, registers.
  • Prebuilt-XBE projects — debug an existing .xbe with no build step.
  • Settings — every RXDK setting and the RXDK Settings screen.
  • Commands — the RXDK command palette / sidebar commands.

Where things live

RXDK keeps shared data outside your project, in a per-user data root, so multiple projects reuse one copy. The data root is platform-specific:

OSData root
Windows%ProgramData%\RXDK (e.g. C:\ProgramData\RXDK)
macOS~/Library/Application Support/RXDK
Linux$XDG_DATA_HOME/rxdk (default ~/.local/share/rxdk)

Inside the data root:

SubfolderContents
sdk/RXDK-SDK include + lib (headers and import/static libs).
tools/Host tools: imagebld, xdvdfs, xbcp, xboxdbg-bridge, xbwatson.
docs/The RXDK-Docs repo clone: xboxsdk/ (API reference) and rxdk/ (this documentation).

Override the SDK/tools locations with the rxdk.stagedSdkPath / rxdk.stagedToolsPath settings. Documentation is delivered by cloning the RXDK-Docs repository (like the SDK libraries); the repo's VERSION file tracks the installed docs version.
