rxdk.project.json — Project Manifest Reference

Every RXDK project has an rxdk.project.json file at its root. The extension locates a project by searching the workspace for this file; it drives the build (which sources/libraries to compile), the PE→XBE image step (imagebld switches), and deploy (what to copy to the console).

There are two kinds of project:

  • Source project — RXDK compiles your sources and links libraries into an .xbe.
  • Prebuilt-XBE project — you point at an existing .xbe (+ optional .pdb/.map); RXDK skips the build and only deploys + debugs. Set the prebuilt object for this mode (see Prebuilt-XBE projects).

Minimal example

{
  "name": "my-triangle",
  "sources": ["src/main.cpp"],
  "libraries": ["libd3d8", "libd3dx8", "libxapi", "libc", "libcpp", "libkernel"],
  "outputDir": "out",
  "imageBuild": {
    "stackSize": 65536,
    "debug": true,
    "noLogo": true,
    "noLibWarn": true
  }
}

Full example (media + embedded file + extra include/defines)

{
  "name": "my-tone",
  "sources": ["src/main.cpp", "src/audio.cpp"],
  "libraries": ["libdsound", "libxapi", "libc", "libcpp", "libkernel"],
  "includePaths": ["include"],
  "defines": ["MY_FEATURE=1", "AUDIO_DEBUG"],
  "outputDir": "out",
  "deployPaths": ["media"],
  "embed": [{ "path": "media/dsstdfx.bin", "name": "dsstdfx" }],
  "imageBuild": {
    "stackSize": 65536,
    "debug": true,
    "noLogo": true,
    "noLibWarn": true,
    "limitMemory": false,
    "noPreload": [".data"]
  }
}

Top-level properties

name

Type: string · Required

The project/title name. Used as the output .xbe/.exe base name and as the remote folder under xe:\ on the console (xe:\<name>\). Use a filesystem-safe name (letters, digits, _, -; start with a letter).

sources

Type: string[] · Source projects only

Project-relative paths to the .c / .cpp files to compile. Globs are not expanded — list each file. A source ending in .cpp/.cxx/.cc marks the project as C++ (pulls in the C++ toolchain and IntelliSense configuration). Example: ["src/main.cpp", "src/render/mesh.cpp"]

libraries

Type: string[] · Source projects only

RXDK-SDK libraries to link, without the .lib extension. Order can matter for static linking; keep the low-level libs (libc, libcpp, libkernel) last. Common libraries:

LibraryProvides
libd3d8Direct3D 8 (graphics)
libd3dx8D3DX 8 math/texture helpers
libdsoundDirectSound (audio)
libxapiXbox API (threads, files, input plumbing, launch)
libxmvXMV video playback
libcC runtime (picolibc)
libcppC++ runtime
libkernelxboxkrnl import stubs (always required)

includePaths

Type: string[] · Optional

Extra project-relative include directories, passed to the compiler as /I after the SDK's include. Use for your own headers. Example: ["include", "third_party/include"]

defines

Type: string[] · Optional

Extra preprocessor defines (/D), appended after the RXDK defaults. Use NAME or NAME=value. Example: ["NDEBUG", "TITLE_ID=0x1234"]

outputDir

Type: string · Optional (default: out)

Project-relative directory for build artifacts (.exe, .xbe, .pdb, .map, intermediate objects).

deployPaths

Type: string[] · Optional

Project-relative directories copied recursively to the console on deploy, preserving structure, under xe:\<name>\. Use for runtime assets your title loads (textures, sounds, video). Example: ["media", "data/levels"] → xe:\my-title\media\….

deployPaths vs embed: deployPaths copies loose files to the console filesystem (read at runtime from disc/HDD). To bake a file into the XBE image instead, use embed.

embed

Type: RxdkEmbedFile[] · Optional

Files embedded inside the XBE at build time via imagebld /insertfile. Retrieve them at runtime by the given resource name. Typical use: the DirectSound standard-effects blob (dsstdfx.bin). Each entry:

  • path — string, required. Project-relative path to the file to embed.
  • name — string, required. Resource name the title uses to locate the embedded data.

Example: [{ "path": "media/dsstdfx.bin", "name": "dsstdfx" }]

imageBuild

Type: RxdkImageBuildOptions · Optional

Switches for the PE→XBE step (imagebld). Omitted keys use imagebld defaults — see Image build options.

prebuilt

Type: RxdkPrebuiltConfig · Optional — presence switches the project to prebuilt-XBE mode

When set (with a valid xbe), RXDK does not compile. sources/libraries/imageBuild are ignored. See Prebuilt-XBE projects and prebuilt fields.

Image build (imageBuild) options

All keys are optional; omit a key to use the imagebld default.

KeyTypeimagebld switchMeaning
stackSizenumber/stack:<bytes>Title stack size in bytes (e.g. 65536).
debugboolean/debugProduce a debug XBE (needed for source-level debugging).
noLogoboolean/nologoSuppress the imagebld banner.
noLibWarnboolean/nolibwarnSuppress library-mismatch warnings.
limitMemoryboolean/limitmemRun with 64 MB (retail) instead of 128 MB (devkit). Use to catch retail-memory bugs.
dontModifyHardDiskboolean/dontmodifyharddiskDon't let the image alter the HDD.
dontMountUtilityDriveboolean/dontmountutilitydriveDon't mount the utility (Z:) drive.
formatUtilityDriveboolean/formatutilitydriveFormat the utility drive on launch.
utilityDriveClusterSizenumber—Cluster size: 16384, 32768, or 65536. Omit/0 = default.
noPreloadstring[]/nopreload:<section>Sections not preloaded at launch (one switch per name), e.g. [".data"].
Recommended for debugging: keep "debug": true. Without it there are no symbols to set breakpoints or inspect variables against.

prebuilt fields

Used only in prebuilt-XBE projects (see Prebuilt-XBE projects).

KeyTypeRequiredMeaning

Notes & gotchas

  • One manifest per project. The extension uses the first rxdk.project.json it finds. Keep one project per workspace folder.
  • sources are not globbed. Add new files to the array explicitly.
  • Strict JSON. No // comments or trailing commas.
  • After editing the manifest, re-run RXDK: Build (and Deploy) so new sources/assets take effect.
