Prebuilt-XBE Projects

A prebuilt-XBE project points RXDK at an existing .xbe (and optional symbols) instead of compiling from source. RXDK skips the build entirely and only deploys and debugs. Use this to debug a title built elsewhere, a legacy XBE, or a binary you don't have full sources for.

Create one

Run RXDK: New Prebuilt XBE Project. Point it at the .xbe (and a .pdb/.map if you have them). It generates an rxdk.project.json with a prebuilt block and a workspace with a debug launch config.

Manifest shape

{
  "name": "my-legacy-title",
  "prebuilt": {
    "xbe": "C:\\builds\\game\\default.xbe",
    "pdb": "C:\\builds\\game\\default.pdb",
    "map": "C:\\builds\\game\\default.map",
    "srcRoot": "C:\\builds\\game\\src",
    "remoteName": "my-legacy-title"
  }
}

See the prebuilt field reference for every key. Only xbe and remoteName are required.

What works

ActionPrebuilt project
BuildNo-op (nothing to compile).
DeployCopies the .xbe to xe:\<remoteName>\.
Run / DebugLaunches on the console; debugging attaches if a .pdb is provided.

Symbols & source

  • pdb enables source-level debugging: breakpoints, Locals, and Globals. Without it you can still launch, but there's nothing to inspect.
  • map is a fallback source of globals when no PDB is available (less detail than a PDB).
  • srcRoot helps map a PDB built on another machine back to your local source tree, so breakpoints bind to the right files.
  • exe (the host PE) is used only to determine image size; RXDK falls back to the XBE header if it's absent.

Refresh source folder

RXDK: Refresh Source Folder re-syncs the workspace's view of the source tree for a prebuilt project (useful after moving or updating the referenced sources).
