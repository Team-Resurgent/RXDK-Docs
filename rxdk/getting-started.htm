Getting Started

From a fresh install to a title running (and debugging) on a devkit.

1. Install prerequisites

Open the RXDK sidebar (Activity Bar → RXDK icon). If setup is incomplete you'll see Setup required → Complete RXDK setup… (command: RXDK: Complete Setup). It installs:

  • .NET 8 runtime — used by the host tools and the debug bridge.
  • RXDK-SDK — include + lib, cloned to the data root's sdk/.
  • Host tools — imagebld, xdvdfs, xbcp, xboxdbg-bridge, xbwatson, in tools/.
  • Zig — the C/C++ compiler toolchain.
  • Documentation — the RXDK-Docs repo (Xbox SDK API reference + this extension guide), cloned to docs/.

The data root is %ProgramData%\RXDK on Windows, ~/Library/Application Support/RXDK on macOS, and ~/.local/share/rxdk on Linux — see Overview → Where things live.

2. Point at your devkit

  • Command: RXDK: Set Xbox IP / Hostname, or the Devkit node in the sidebar.
  • Windows: if left blank, RXDK reads the Xbox SDK registry (XBSetIP / Neighborhood).
  • macOS/Linux: set rxdk.defaultConsole in settings (no registry). You can also set it in the RXDK Settings screen.

The devkit must be on and reachable on the network.

3. Create a project

RXDK: New Project opens the wizard. Pick a template, name, and parent folder:

TemplateShows
D3D8 TriangleRotating Direct3D 8 triangle
DSound ToneDirectSound audio
XInput GamepadController input
XMV PlayVideo playback

The wizard scaffolds the project (including its rxdk.project.json), a VS Code workspace, and build/debug tasks, then opens it.

4. Build

RXDK: Build compiles sources, links libraries, and runs imagebld to produce the .xbe (under outputDir, default out/). Output streams to a terminal + the RXDK output channel.

5. Deploy & run

  • RXDK: Deploy copies the .xbe and any deployPaths to xe:\<name>\.
  • RXDK: Run launches the deployed title.

(Deploy and run require the .NET runtime and a reachable console.)

6. Debug

Press F5 (or RXDK: Debug). This builds, deploys, launches under the debugger, and connects the symbol bridge. Set breakpoints and inspect state — see Debugging.

Testing checklist

A quick loop for verifying a change on hardware:

  1. Edit code → RXDK: Build (watch the terminal for compile/link errors).
  2. RXDK: Deploy (confirm files land under xe:\<name>\).
  3. F5 to debug, or RXDK: Run to just launch.
  4. Set a breakpoint in main → confirm it binds (solid dot) and hits.
  5. Inspect Locals and Globals (see the Globals visibility toggle).
  6. Watch title output in the Xbox Title output channel (toggle xbox.showTitleOutput), or launch xbWatson.

Troubleshooting

SymptomTry
"No rxdk.project.json found"Open the project folder as the workspace root.
"RXDK SDK not installed"Reload the window (triggers the clone), or run Complete Setup.
Deploy/run fails immediatelyCheck the devkit IP (Set Xbox IP), power, and network.
Breakpoints don't bindEnsure imageBuild.debug is true and you rebuilt/redeployed.
No symbols / empty LocalsConfirm the .pdb sits next to the .xbe in outputDir.
".NET runtime missing"Run RXDK: Install .NET 8 Runtime.
